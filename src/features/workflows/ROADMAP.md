# 工作流画布前端推进说明

## 当前定位

工作流画布当前处于 **核心原型向真实路径验证过渡** 阶段。

本模块已经具备基础编辑器交互：

- 节点库与拖拽新增节点
- React Flow 画布、节点、连线、缩放与视图适配
- 开始节点、LLM 节点、结束节点的基础展示
- 节点选中、删除保护、撤销/重做
- 连线规则校验：禁止自连、重复连线、成环
- LLM/HTTP 节点基础配置面板
- 暗色/亮色主题适配
- 工作流 JSON 输出

但当前仍属于前端原型阶段，尚未进入完整编辑器产品化阶段。

---

## 本周重点目标

### 目标

本周不追求完整工作流编辑器能力，重点是跑通一条最小真实路径：

```text
开始节点 -> LLM 节点 -> 结束节点
```

该路径需要能支撑内部演示和后续接口设计讨论。

### 本周范围

1. 固定最小节点链路
   - 开始节点作为流程入口
   - LLM 节点作为核心处理节点
   - 结束节点作为流程出口
   - 保证三类节点在画布上的展示、连线、选中和配置体验稳定

2. 完善 LLM 节点配置
   - 模型选择
   - temperature
   - 最大输出 token
   - 系统提示词
   - 用户输入/提示词主体字段（如需要）
   - 输出字段说明

3. 明确前端保存结构
   - 输出稳定的 `nodes + edges` JSON
   - 节点 config 字段结构清晰
   - 去除不适合保存的临时 UI 字段
   - 为后续后端保存接口预留数据形态

4. 做前端模拟运行态
   - 点击运行后按 `开始 -> LLM -> 结束` 顺序高亮
   - 节点状态至少包含：待执行、运行中、成功、失败
   - 连线可在运行时展示流动态
   - 展示简单运行日志或结果摘要

5. 保持基础编辑体验稳定
   - 删除、撤销/重做、视图适配可用
   - 开始/结束节点不可删除
   - 暗色模式视觉正常
   - 控制台无明显运行时错误

### 本周不做

以下内容本周不作为交付重点，避免范围扩大：

- 完整节点体系
- 复杂条件分支
- 变量系统
- 插件、MCP、知识库等真实能力闭环
- 自动布局
- 复制粘贴、右键菜单、批量选择
- 大流程性能优化
- 真实接口联调

---

## 本周验收标准

本周完成后，应能演示以下流程：

1. 打开工作流页面
2. 看到默认的 `开始 -> LLM -> 结束` 流程
3. 选中 LLM 节点并修改配置
4. 点击保存，得到结构清晰的工作流 JSON
5. 点击运行，前端模拟执行完整链路
6. 运行过程中节点和连线状态可视化
7. 结束节点展示最终结果或摘要

## 后续路线

### 阶段 1：可演示版本增强

目标：让内部评审能完整理解产品方向。

工作项：

- 完善最小路径运行态
- 增加本地草稿保存与恢复
- 增加导出/导入 JSON
- 补齐关键节点的占位配置
- 统一节点、连线、面板的视觉状态

### 阶段 2：可试用版本

目标：支持产品、前端、后端围绕真实数据结构评审。

工作项：

- 拆分节点配置面板
- 建立节点默认配置与类型定义
- 增加更多节点类型：
  - 知识库
  - 指定回复
  - 条件判断
  - 代码执行
  - 插件
  - MCP 工具
- 优化撤销/重做策略
- 增加运行日志面板
- 完善空状态、错误态、选中态和 hover 态

### 阶段 3：产品级前端基座

目标：形成可长期扩展的工作流编辑器模块。

工作项：

- 节点 schema 化
- 复杂节点配置能力
- 变量引用与输出映射
- 条件分支与多出口连线
- 复制粘贴、右键菜单、快捷键
- 自动布局
- 大流程性能优化
- 与后端接口结构对齐

---

## 当前风险

1. 节点 config 结构仍偏原型，后续需要尽早稳定。
2. 非 LLM 节点配置面板仍是占位。
3. 保存输出中可能包含 React Flow 测量字段，需要在正式保存前清洗。
4. 运行态尚未形成独立状态模型。
5. 画布编辑器类功能复杂度高，应避免一次性追求完整能力。

---

## 推进原则

1. 本周只围绕最小真实路径交付。
2. 优先保证演示链路闭环，再扩展节点种类。
3. 前端数据结构尽量贴近未来接口，但不提前过度设计。
4. 每次新增节点能力时，同步补充类型、默认配置和配置面板。
5. UI 继续遵循 AI Flow Admin 现有组件规范和设计 token。
